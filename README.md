# resend-client

> A self-hosted, open-source, multi-tenant email client for [Resend](https://resend.com) — built entirely on Cloudflare's edge platform.

[![Deploy to Cloudflare](https://deploy.workers.cloudflare.com/button)](https://deploy.workers.cloudflare.com/?url=https://github.com/abdul-karim-mia/resend-client)

## Features

- 📥 **Multi-tenant inbox** — manage multiple Resend accounts/domains in one place
- 🧵 **Email threading** — RFC 2822 `In-Reply-To` / `References` header resolution
- 🤖 **Workers AI** — draft replies, summarize threads, adjust tone (formal / casual / concise), custom free-form prompts
- ⚡ **Quick Reply chips** — 3 AI-generated one-click replies per email, with ✏ Edit-before-send
- ✏️ **Rich composer** — contenteditable editor, CC/BCC, Gmail-style reply quoting, drag-and-drop attachments, draft auto-save
- 🔒 **Secure by default** — JWT auth, AES-256-GCM encrypted API keys, sandboxed email rendering, HMAC webhook verification
- 🔎 **Full-text search** — FTS5 SQLite search across all email fields
- 📋 **Resend templates** — first-class template support with variable interpolation (SDK v4)
- ⌨️ **Keyboard shortcuts** — Gmail-style (C, R, E, #, ?, ⌘K)
- 📎 **Attachments** — drag-and-drop upload to R2, inline image embedding
- 🌍 **Edge-native** — ~30ms global cold start, no servers to manage

## Deploy to Cloudflare (One-Click)

Click the **Deploy to Cloudflare** button above — Cloudflare will:

1. Fork this repo to your GitHub account
2. Show a setup form — create your D1 database and R2 bucket with one click
3. Prompt you for 4 required secrets (see below)
4. Build and deploy automatically — **no CLI or manual steps needed**

> **⚡ Zero SQL needed** — the database schema is initialized automatically on the first request.

### Secrets Required at Deploy Time

Run the setup script locally to generate all values:

```bash
node scripts/setup-secrets.mjs
```

It will print all four values. Paste them into the Cloudflare deploy form:

| Secret | Description |
|---|---|
| `MASTER_ENCRYPTION_KEY` | AES-256 key (auto-generated by setup script) |
| `JWT_SECRET` | Signs admin session tokens (auto-generated) |
| `ADMIN_USERNAME` | Your login username (e.g. `admin`) |
| `ADMIN_PASSWORD_HASH` | SHA-256 hash of your password (auto-generated) |

## GitHub Actions Auto-Deploy

Every push to `main` automatically builds and deploys via `.github/workflows/deploy.yml`.

### One-Time Setup (6 secrets)

Go to **GitHub repo → Settings → Secrets & variables → Actions** and add:

| Secret | Where to get it |
|---|---|
| `CLOUDFLARE_API_TOKEN` | [Cloudflare → My Profile → API Tokens](https://dash.cloudflare.com/profile/api-tokens) → use *"Edit Cloudflare Workers"* template |
| `CLOUDFLARE_ACCOUNT_ID` | Cloudflare dashboard — right sidebar |
| `MASTER_ENCRYPTION_KEY` | `node scripts/setup-secrets.mjs` |
| `JWT_SECRET` | `node scripts/setup-secrets.mjs` |
| `ADMIN_USERNAME` | Your chosen admin username |
| `ADMIN_PASSWORD_HASH` | `node scripts/setup-secrets.mjs` |

After adding these secrets, every push to `main` deploys with **zero manual steps**.

> The workflow also type-checks both `frontend` and `worker` before deploying — broken TypeScript won't reach production.

## Manual Deploy (CLI)

If you prefer the command line:

### 1. Clone & Install

```bash
git clone https://github.com/abdul-karim-mia/resend-client.git
cd resend-client
pnpm install
```

### 2. Generate Secrets

```bash
node scripts/setup-secrets.mjs
# Creates .dev.vars for local dev and prints production JSON
```

### 3. Create Cloudflare Resources

```bash
# Create D1 database
npx wrangler d1 create resend-client-db
# → Copy the database_id printed into wrangler.jsonc

# R2 bucket is auto-provisioned on first deploy
```

> **No schema step** — tables are created automatically on the first request.

### 4. Push Secrets to Cloudflare

```bash
# Push all 4 secrets at once (paste the JSON printed by setup-secrets.mjs)
echo '{"MASTER_ENCRYPTION_KEY":"...","JWT_SECRET":"...","ADMIN_USERNAME":"...","ADMIN_PASSWORD_HASH":"..."}' \
  | npx wrangler secret bulk --env="" --config wrangler.jsonc
```

### 5. Build & Deploy

```bash
pnpm deploy  # builds frontend + deploys worker + assets
```

Your app is live at `https://resend-client-worker.<your-subdomain>.workers.dev`

### 6. Configure Resend Webhooks

1. Sign in → **Admin** → **+ Add Account**
2. Copy the **Webhook URL** and **Webhook Secret**
3. In Resend dashboard → **Webhooks** → Add endpoint
4. Subscribe to: `email.received`, `email.sent`, `email.delivered`, `email.bounced`, `email.opened`

See [docs/setup.md](docs/setup.md) for detailed DNS/MX configuration.

## Local Development

```bash
# Generate secrets → writes .dev.vars automatically
node scripts/setup-secrets.mjs

# Start worker on :8787 + frontend on :5173 (with proxy)
pnpm dev
```

Open `http://localhost:5173` — API calls are proxied to the Worker.

> **No schema step needed** — D1 tables are auto-created on the first request to the local Worker.

## Custom Domain

1. Cloudflare dashboard → **Workers & Pages** → `resend-client-worker`
2. **Settings** → **Domains & Routes** → **Add Custom Domain**
3. Enter your domain (e.g. `mail.yourdomain.com`) — SSL is automatic
4. Update webhook URLs in Resend to use the custom domain

## Changing Your Password

```bash
# Re-run setup (backs up existing .dev.vars)
node scripts/setup-secrets.mjs

# Push new hash to production
echo '{"ADMIN_PASSWORD_HASH":"<new-hash>"}' \
  | npx wrangler secret bulk --env="" --config wrangler.jsonc
```

## Architecture

```
Browser ──► Cloudflare Worker (Hono.js)
                ├── /api/auth/*         JWT authentication
                ├── /api/emails/*       Email CRUD + FTS5 search
                ├── /api/send/*         Outbound via Resend SDK + draft save
                ├── /api/ai/*           Workers AI — draft, summarize, tone, custom prompt, quick-reply
                ├── /api/attachments/*  R2 upload/download/inline
                ├── /api/templates/*    Resend template proxy + variable resolution
                ├── /api/settings/*     Account + sender identity management
                ├── /webhook/:id/inbound    Inbound email processing
                ├── /webhook/:id/events     Delivery status updates
                └── /*                 React SPA (Vite build, served from R2)

                ├── D1 (SQLite + FTS5)  emails, accounts, attachments, templates, senders
                ├── R2                  Email attachment files
                └── Workers AI          Llama 3.2, DeepSeek R1 for AI features
```
