# Setup Guide

## Prerequisites

- Cloudflare account (free tier works fine)
- Resend account with at least one verified domain
- Node.js 20+ and pnpm 9+

---

## Option A — One-Click Deploy (No CLI)

Click the **Deploy to Cloudflare** button in the [README](../README.md):

1. Cloudflare forks the repo and shows a setup form
2. Create your D1 database and R2 bucket with one click
3. Paste the secrets generated by `node scripts/setup-secrets.mjs`
4. Click **Deploy** — done

---

## Option B — Manual CLI Deploy

### Step 1 — Clone & Install

```bash
git clone https://github.com/abdul-karim-mia/resend-client.git
cd resend-client
pnpm install
```

---

### Step 2 — Generate Secrets

```bash
node scripts/setup-secrets.mjs
```

This script:
- Generates `MASTER_ENCRYPTION_KEY` and `JWT_SECRET` (cryptographically random)
- Prompts for admin username + password → SHA-256 hashes the password
- Writes all values to `.dev.vars` (for local dev)
- Prints the production JSON for `wrangler secret bulk`

---

### Step 3 — Create D1 Database

```bash
npx wrangler d1 create resend-client-db
```

Copy the `database_id` from the output and update `wrangler.jsonc` at the repo root:

```jsonc
"d1_databases": [
  {
    "binding": "DB",
    "database_name": "resend-client-db",
    "database_id": "YOUR_NEW_ID_HERE"   // ← replace this line
  }
]
```

> **No schema step needed** — D1 tables are created automatically on the first Worker request.

> **R2 bucket** — automatically provisioned on first `wrangler deploy`. No manual creation needed.

---

### Step 4 — Push Secrets to Cloudflare

Paste the JSON printed by the setup script:

```bash
echo '{"MASTER_ENCRYPTION_KEY":"...","JWT_SECRET":"...","ADMIN_USERNAME":"...","ADMIN_PASSWORD_HASH":"..."}' \
  | npx wrangler secret bulk --env="" --config wrangler.jsonc
```

Or push one at a time:

```bash
npx wrangler secret put MASTER_ENCRYPTION_KEY --env="" --config wrangler.jsonc
npx wrangler secret put JWT_SECRET             --env="" --config wrangler.jsonc
npx wrangler secret put ADMIN_USERNAME         --env="" --config wrangler.jsonc
npx wrangler secret put ADMIN_PASSWORD_HASH    --env="" --config wrangler.jsonc
```

---

### Step 5 — Build & Deploy

```bash
pnpm deploy
```

This runs `pnpm build` (Vite frontend) then `wrangler deploy` (Worker + assets).

Your app is live at `https://resend-client-worker.<your-subdomain>.workers.dev`

---

## Step 6 — Add a Resend Account (per domain)

1. Open the deployed app and sign in
2. Navigate to **Admin → + Add Account**
3. Fill in:
   - **Name** — e.g. `Support Inbox`
   - **Domain** — e.g. `support.yourdomain.com`
   - **From Name** — e.g. `Support Team`
   - **From Email** — e.g. `support@yourdomain.com`
   - **Resend API Key** — from [Resend dashboard → API Keys](https://resend.com/api-keys)
4. Click **Create** — you'll receive:
   - **Inbound Webhook URL** — copy this
   - **Webhook Secret** — copy this

---

## Step 7 — Configure Resend Inbound Email

### DNS Setup (MX Record)

In your DNS provider, add an MX record for the subdomain you want to receive email on:

| Type | Host | Value | Priority |
|---|---|---|---|
| MX | `support` (or `@`) | `inbound.resend.com` | 10 |

> This routes incoming email for your domain through Resend's inbound processing pipeline.

Allow 24–48 hours for DNS propagation.

### Resend Webhook

In the [Resend dashboard](https://resend.com):

1. Go to **Webhooks** → **+ Add Endpoint**
2. **URL** — the Inbound Webhook URL from Step 6
3. **Secret** — the Webhook Secret from Step 6
4. **Events** to subscribe:
   - `email.received` (inbound emails trigger your inbox)
   - `email.sent`
   - `email.delivered`
   - `email.bounced`
   - `email.opened`

Repeat Steps 6–7 for each domain/account.

---

## Step 8 — Custom Domain (Optional)

1. Cloudflare dashboard → **Workers & Pages** → `resend-client-worker`
2. **Settings** → **Domains & Routes** → **Add Custom Domain**
3. Enter your subdomain (e.g. `mail.yourdomain.com`) — SSL is automatic
4. Update your Resend webhook URLs to use the custom domain

## Local Development

```bash
# Generate secrets + write .dev.vars automatically
node scripts/setup-secrets.mjs

# Start both servers (worker :8787, frontend :5173 with proxy)
pnpm dev
```

Frontend: `http://localhost:5173`
Worker API: `http://localhost:8787`

The local Worker connects to your Cloudflare D1 remotely (or you can run `pnpm db:apply` to get a local copy).

> **No schema step** — D1 tables are auto-initialized on the first request.
